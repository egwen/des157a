Overlays Design

Best Practices for Overlays

7 Feburary 2023

An overlay is an element that displays on top of the main window's content. Overlays are a way to simplify a webpage's UI and draw the user's focus to important information and interactions. Overlays typically disable the entire window and require user interaction to proceed. However, since overlays force the user to do something, if they are not used in the appropriate context with the proper layout, they can become tedious and annoying for users.

The appropriate purpose of an overlay would be to show the user important or additional information. The best overlays only have vital information or prompt the user to input information required for the website's functionality. An example of appropriate content for an overlay would be a user's login and password, since that information is typically required for a website to fully function. Another appropriate context would be to provide additional content or information to the user without leaving the original page. For example, an overlay can be used to enlarge an image or video without leaving the original page. An overlay should have appropriate content to reduce the likelihood that users immediately dismiss the element or leave the page.

Designers should also consider effective UX principles so that users are more likely to interact with the overlay. The overlay should have a descriptive title so that the user understands the purpose of the element and its relation to the original page. Any buttons on the overlay should have labels that clearly reflect the purpose of the element. Since overlays disable the entire page, users should also be able to easily close the element by either a mouse or keyboard interaction. Overlays should also implement web accessibility principles like keyboard accessibility and ARIA tags. An overlay's design should be clear and understandable to users to reduce frustration and the likelihood that they dismiss the overlay completely.

Form Design

Best Practices for Form Design

31 January 2023

Forms are a primary method for websites to interact with users, so it is essential that forms are simple and straightforward, following the concept of “Less is More”. A form's simple design reflects the amount of information and effort required of the user and increases the likelihood that they interact with and complete the form. Both the quantity and quality of the form's inputs determines the usability of the form.

The overall form's structure should have minimal content and keep a simple layout to increase usability. The most basic approach to simplify a form is to minimize the number of inputs. However, in cases where the form must be long, designers should visually communicate the number of steps required so users know what to expect and are more likely to complete the form. The form should also progress from simple to complex questions so that users are less likely to abandon the form at the very beginning. The layout of the form should also maintain vertical visual flow with single columns. This reduces both the amount of vertical scrolling and avoids horizontal interruptions to the user's progress.

Each individual input field should also have a simple design with enough information for the user to understand the expected input. Labels and placeholder text are crucial components of information so users can fill out the form quickly without any mistakes. They should be placed close to the input field and should persist as the user types, especially for longer forms. Designers should also consider input type to optimize user navigation speed. Inputs that require multiple interactions can become tedious for users, decreasing usability.

An webite that follows some of these practices is Duolingo, which uses forms in their lessons. The form uses select input types to avoid user frustration and progress bars to indicate the length of the form, increasing the likelihood that learners complete their lesson.
